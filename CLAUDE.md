# CLAUDE.md — Corpus research-synthesis assistant

Guidance for working in this repo. Project-specific facts only; general coding
rules come from the user's global config.

## What this is

A Consensus/Elicit-style medical **research-synthesis** app: a Paper Search page
and a Research Report (PRISMA funnel + consensus meter + GRADE evidence grading +
cited synthesis). Three runtimes behind one stable API contract:

- **Frontend** — static, dependency-free vanilla JS (`index.html`, `report.html`,
  `js/`). No bundler, no framework. Calls the API over `fetch` + `EventSource`.
- **Server** — Fastify + TypeScript mock gateway (`server/`), no DB (in-memory).
- **Brain** — Python synthesis worker (`synthesis/`), spawned by the server.

## The one invariant: the contract is the seam

`api/types.ts` is the **single source of truth**. Three consumers depend on it:
the browser renderer (`js/api.js` `CorpusRender`), the mock server, and the Python
brain (`synthesis/corpus_brain/contract.py` mirrors it). **Before changing
`ContentBlock` or `ReportEvent`, account for all three.** The renderer ignores
unknown block types on purpose, so new figures ship backend-first without breaking
the frontend. A report is an ordered `ContentBlock[]` + a citation layer, driven by
one `ResearchQuery`.

The SSE event stream (`status → funnel → meter → block… → references → done`) is
the same whether generated by the fixture or the brain — keep it that way.

## Repository-wide pattern: abstraction + mock default + real adapter behind a flag

This shape repeats in **three** places (same idea each time — learn it once):

| Seam | Interface | Default (offline) | Real adapter | Selector |
|---|---|---|---|---|
| Literature sources | `providers/provider.ts` | `MockProvider` | OpenAlex / S2 / bioRxiv / PubMed | `providers/registry.ts` (`CORPUS_PROVIDERS`) |
| LLM backend | `synthesis/.../llm/base.py` | `StubLLM` | `AnthropicLLM` | `make_client` (`ANTHROPIC_API_KEY`) |
| Job queue | `server/src/queue/types.ts` | `MemoryQueue` | `BullMqQueue` (Redis) | `queue/registry.ts` (`QUEUE_DRIVER`) |

**Adding a source = implement the interface + add one registry line.** Real
adapters lazy-load heavy deps so the default path needs nothing extra.

## Running

```bash
# Frontend (static) + server + (optional) brain
cd server && npm install && npm run mock          # :8787  /api/v1
python3 -m http.server 8731                        # repo root → :8731
USE_BRAIN=1 CORPUS_BRAIN_STUB=1 npm run mock        # route /reports through the offline brain
```

Point pages at a backend via `window.CORPUS_API_BASE` (default
`http://localhost:8787/api/v1`) or `?api=`.

## Testing — everything runs offline, no network or API key

| Where | Command | Covers |
|---|---|---|
| repo root | `npm install && npm test` | Vitest+jsdom unit (`readQuery`, every `CorpusRender.block`) + Playwright e2e |
| `server/` | `npm test` | Vitest: providers, queue (concurrency/retry/dedup), hub (replay), scheduler |
| `synthesis/` | `python3 -m pytest` | Pipeline stages, orchestrator, citation guardrail, budget, eval |
| `synthesis/` | `python3 eval_gate.py` | **Release gate** — grounding metrics over the gold set |

Offline is a **hard convention**: e2e stubs Google Fonts and runs serial
(`workers:1`) against the one stateful mock; providers use recorded HTTP fixtures
(`server/fixtures/http/`); the brain uses `StubLLM` (`CORPUS_BRAIN_STUB=1`). Don't
introduce tests that need the network or a key.

## Things not to break

- **Citation grounding guardrail** (`synthesis/.../synthesize.py`): every
  `{{cite:N}}` must resolve to a real `Reference` or it is stripped. This is the
  medical-safety spine — don't bypass it.
- **`eval_gate.py` is a release blocker** for the Brain, not a nice-to-have.
- **Model ids**: consult the `claude-api` skill; never guess. Current tiers —
  `claude-haiku-4-5` (screen), `claude-sonnet-4-6` (extract), `claude-opus-4-8`
  (synthesize). Adaptive thinking on Sonnet/Opus; no thinking/effort on Haiku.
- **SSE ids**: the hub assigns monotonic event ids; clients reconnect via
  `Last-Event-ID` and replay only what they missed. Keep the id sequence intact.
- **The brain is a subprocess** speaking NDJSON `ReportEvent`s on stdout — the
  worker must not print stray stdout (the Node side parses it line by line).
- Stored entities are replaced wholesale, never field-mutated (see `store.ts`).

```

index.html report.html  frontend pages
js/                     api.js (client+renderer), report-view.js (stream+render),
                        app.js (report page), paper-search.js (search drawer), config.js

  jobs.ts               enqueue + worker: run generation, publish to hub
  hub.ts                per-report event log + Last-Event-ID pub/sub
  queue/                JobQueue + MemoryQueue + BullMQ + registry
  scheduler.ts          weekly recompute
  brain.ts              spawns the Python worker, relays NDJSON
  providers/            LiteratureProvider + adapters + aggregator + registry
  pipeline/             fixture report + references + stream
synthesis/              the Brain (Python)
  corpus_brain/         contract, budget, llm/, pipeline/ (screen→extract→grade→synthesize), eval/
  run.py                stdin job spec → stdout NDJSON events
  eval_gate.py          CI release gate

```
